Why Git & GitHub Matter

  • Version control: track changes over time
  • Collaboration: work in parallel without overwriting each other
  • Open-source: contribute to projects globally
  • Portfolio: showcase your work to employers
Think of Git as a time machine for your code and GitHub as the global workspace where you share and collaborate.

Git vs. GitHub

Git

Local version control system

Git = Engine

GitHub

Cloud-based hosting + collaboration platform

GitHub = Garage & showcase space

Developer's laptop → Git → GitHub → Team collaboration

Setting Up

  1. Install Git: https://git-scm.com
  2. Create GitHub account: https://github.com
  3. Configure Git:
    git config --global user.name "Your Name"
    git config --global user.email "you@example.com"
  4. Clone starter repo:
    git clone https://github.com/example/starter-repo.git
Before we can code, we need to ensure Git is ready to track changes and GitHub is set up to host them.

Git Basics: Commits

git init → Start tracking a project
git add → Stage changes
git commit → Save a snapshot

Commit message tips:

  • Short summary (~50 chars)
  • Use imperative mood: Add feature, not Added feature

Example:

git add index.js
git commit -m "feat: add user authentication"

Branching & Merging

Branch = Separate line of work

Main branch = Always stable

Create & switch branch:

git branch feature-login
git checkout feature-login

Merge when work is ready

Pull Requests

  • Request to merge code into main
  • Enables code review & discussion

Best practices:

  • Small, focused changes
  • Clear title & description

Demo Flow:

  1. Push branch to GitHub
  2. Open PR
  3. Review & merge

READMEs

Why they matter:

  • First impression of your repo
  • Help others understand/use your code

Include:

  • Project description
  • Installation
  • Usage examples
  • License

Example Markdown:

# My Project
A simple Node.js app...

## Installation
npm install

## Usage
npm start

Issues & Project Boards

Issues: Track bugs, ideas, tasks

Project boards: Kanban for task tracking

Link commits to issues with:

git commit -m "fix: correct typo in login #12"

Demo: Create issue → Add to board → Link commit

GitHub Pages Deployment

  1. Go to repo Settings
  2. Enable Pages → Source: main branch
  3. Save & get live URL
  4. Optional: Select a theme

Live Example: https://username.github.io
